# TeamFinder

TeamFinder — веб-приложение на Django для поиска команды под pet-проекты.  
Пользователи могут регистрироваться, создавать проекты, присоединяться к чужим проектам, добавлять проекты в избранное, указывать навыки в профиле и фильтровать участников по навыкам.

Проект выполнен в рамках проектной работы по курсу Ассоциированные программы: backend-разработчик.

## Структура проекта

Основные приложения:

- `users` — пользователи, профили, навыки, регистрация, авторизация;
- `projects` — проекты, участники проектов, избранное;

## Модели

### User

Кастомная модель пользователя.  
Авторизация выполняется по email.

Основные поля:

- `email`
- `name`
- `surname`
- `avatar`
- `phone`
- `github_url`
- `about`
- `skills`
- `is_active`
- `is_staff`

### Skill

Модель навыка пользователя.

Основные поля:

- `name`

### Project

Модель проекта.

Основные поля:

- `name`
- `description`
- `owner`
- `created_at`
- `github_url`
- `status`
- `participants`

### Favorite

Модель избранного проекта.

Основные поля:

- `user`
- `project`
- `created_at`

Для пары `user` + `project` установлено ограничение уникальности.

## Подготовка проекта к запуску

### 1. Клонировать репозиторий

```bash
git clone <ссылка-на-репозиторий>
cd team-finder-ad-main-3
```

---

### 2. Создать виртуальное окружение

```bash
python -m venv venv
```

Активировать окружение:

#### Git Bash / Linux / macOS

```bash
source venv/Scripts/activate
```

#### Windows PowerShell

```powershell
venv\Scripts\Activate.ps1
```

---

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

---

### 4. Создать файл `.env`

В корне проекта нужно создать файл `.env` на основе `.env_example`.

Пример содержимого `.env`:

```env
DJANGO_SECRET_KEY=django-insecure-teamfinder-dev-key
DJANGO_DEBUG=True

POSTGRES_DB=teamfinder
POSTGRES_USER=teamfinder
POSTGRES_PASSWORD=teamfinder
POSTGRES_HOST=localhost
POSTGRES_PORT=5432

TASK_VERSION=2
```

---

### 5. Запустить PostgreSQL через Docker Compose

```bash
docker compose up -d
```

Проверить, что контейнер запущен:

```bash
docker compose ps
```

---

### 6. Применить миграции

```bash
python manage.py migrate
```

---

### 7. Создать суперпользователя

```bash
python manage.py createsuperuser
```

При создании нужно указать:

- email;
- имя;
- фамилию;
- пароль.

---

### 8. Создать демонстрационные данные

В проекте есть management-команда для создания тестовых пользователей, навыков, проектов, участников и избранного.

```bash
python manage.py seed_data
```

После выполнения команды будут созданы демонстрационные данные для быстрой проверки проекта.

---

### 9. Запустить сервер разработки

```bash
python manage.py runserver
```

Проект будет доступен по адресу:

```text
http://127.0.0.1:8000/
```

Административная панель:

```text
http://127.0.0.1:8000/admin/
```

## Особенности реализации

### Кастомный пользователь

В проекте используется собственная модель пользователя `users.User`.  
Вход выполняется по email и паролю.

### Навыки пользователя

Пользователь может добавлять и удалять навыки в своём профиле без перезагрузки страницы.  
Для автодополнения используется endpoint:

```text
GET /users/skills/?q=<query>
```

Добавление навыка:

```text
POST /users/<id>/skills/add/
```

Удаление навыка:

```text
POST /users/<id>/skills/<skill_id>/remove/
```

### Фильтрация участников по навыкам

На странице участников можно выбрать навык и отфильтровать пользователей:

```text
/users/list/?skill=<Название навыка>
```

### Участие в проекте

Авторизованный пользователь может присоединиться к открытому проекту или отказаться от участия.

Endpoint:

```text
POST /projects/<id>/toggle-participate/
```

### Завершение проекта

Владелец проекта может завершить свой проект.

Endpoint:

```text
POST /projects/<id>/complete/
```

После завершения статус проекта меняется на `closed`.

### Избранное

Авторизованный пользователь может добавлять проекты в избранное и удалять их оттуда.

Endpoint:

```text
POST /projects/<id>/toggle-favorite/
```

Страница избранных проектов:

```text
/projects/favorites/
```
